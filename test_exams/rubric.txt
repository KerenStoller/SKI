Computer Science Bagrut Simulation – 5 Units: Grading Rubric
Total Points: 100

Q1 — isBalanced(int[] arr) (30 points)
* Must check if the array length is even, returning false if it is odd.
* Must calculate the sum of the elements in the first half of the array.
* Must calculate the sum of the elements in the second half of the array.
* Must return true if the two sums are exactly equal, and false otherwise.
* Deduct 5 if the even-length validation is missing or incorrect.
* Deduct 10 if loop bounds are incorrect for splitting the array exactly in half.
* Deduct 10 if the summation logic is flawed or missing.
* Deduct 5 if return values are inverted or missing.

Q2 Part A — Team class (15 points)
* Must declare private fields for teamName (String), numMembers (int), and needsComputers (boolean).
* Must have a constructor that initializes all three fields from parameters.
* Must have standard get and set methods for all attributes.
* Deduct 3 per missing field.
* Deduct 5 if the constructor is missing or does not initialize all fields.
* Deduct 5 if getters and setters are missing.
* Deduct 2 if fields are declared public instead of private (encapsulation violation).

Q2 Part B — Hackathon class (25 points)
* Must declare properties: teams (array of Team), maxTeams (int), and currentTeams (int).
* Must have a constructor that receives maxTeams, initializes the array to that size, and sets currentTeams to 0.
* Must have addTeam(Team t) that adds the team if there is room, increments the count, and returns true/false based on success.
* Must have countComputersNeeded() that iterates over the teams and sums numMembers only if needsComputers is true.
* Deduct 5 if the constructor fails to correctly size the array or initialize the counter.
* Deduct 5 if addTeam does not check if the array is full or fails to increment the counter.
* Deduct 5 if countComputersNeeded ignores the needsComputers flag or calculates the sum incorrectly.
* Deduct 3 if methods violate encapsulation by reading private fields of Team directly instead of using getters (if applicable based on Part A).

Q3 — isValidPassword(String pass) (30 points)
* Must be declared as a static method.
* Must check that the string length is at least 8 characters.
* Must check that the string contains at least one uppercase letter ('A'-'Z').
* Must check that the string contains at least one digit ('0'-'9').
* Deduct 5 if the method is not static.
* Deduct 8 per missing condition check (length, uppercase, digit).
* Deduct 5 if the return value is inverted (e.g., returns true for invalid passwords).
* Minor stylistic issues (e.g., using boolean flags vs. early returns) are not deducted.
